1、params传参

优点：刷新页面，参数不丢失

缺点：1.只能传字符串，传值过多url会变得很长 2. 参数必须在路由上配置

是通过在路由后面写/:参数名将路径配合?name=xxx的形式

通过useParams来获取路径参数

路由配置

<Routes>
      <Route path="/" element={<From />} />
      <Route path="to:number" element={<To />} />
    </Routes>

From组件

import React from "react";
import { useNavigate } from 'react-router-dom';
const From = () => {

  const navigate = useNavigate()

  const to =()=>{

  }
  return (<>
  <h1>函数式组件传值</h1>
  <button onClick={()=>{to()}}>传参</button>
  </>)
};

二、类组件

类组件不能用到useNavigate等hooks,所以采用NavLink的方式跳转路由，同理，函数式组件中也可以用NavLink的方式跳转路由

如果不用withRouter，发现组件的this.props是一个空对象

经过各方查阅，我们在react-router-dom v6版本中，已经去除了withRouter，所以在这里我们可以自己封装，但还是推荐使用函数式组件hooks

js版本:

import { useLocation, useNavigate } from "react-router-dom";
export function withRouter( Child ) {
  return ( props ) => {
    const location = useLocation();
    const navigate = useNavigate();
    const params = useParams();
    return <Child { ...props } navigate={ navigate } location={ location } params={ params }/>;
  }
}

ts版本:

import React from "react";
import { NavigateFunction, useLocation, useNavigate, useParams } from "react-router";

export interface RoutedProps<Params = any, State = any> {
    location: State;
    navigate: NavigateFunction;
    params: Params;
}


export function withRouter<P extends RoutedProps>( Child: React.ComponentClass<P> ) {
    return ( props: Omit<P, keyof RoutedProps> ) => {
        const location = useLocation();
        const navigate = useNavigate();
        const params = useParams();
        return <Child { ...props as P } navigate={ navigate } location={ location } params={ params }/>;
    }
}

扩展延伸:react-router-dom中NavLink和Link的区别

在 ReactJS 中，有三种不同的链接。这些是NavLink、、Link和a链接，它们都有不同的用途。

  1. **NavLink**：当您想要突出显示当前或活动链接时使用此选项。这与activeClassName启用它的属性一起使用。请参见下面的示例。
<NavLink to="/home" activeClassName="active" >Home</NavLink><br/>

然后CSS可以根据您在App.css文件中的选择设置样式。让我们将文本颜色设为红色以使其简单。

.active{
  color:red;
}
  1. **Link**：当您的链接没有特殊样式或突出显示时使用。请参见下面的示例。
<Link to="/not-active">Not Active </Link><br/>

注意：当您需要链接到属于您的应用程序的页面时，请使用NavLink或。Link对于外部链接，a是首选。

  1. **锚标记a**：这用于网页外的链接。
    请参见下面的示例。
<a href="https://www.educative.io/edpresso">Visit Edpresso </a>

结论

当NavLink您想要将链接突出显示为活动时使用。因此，在每次路由到页面时，链接都会根据activeClassName. Link用于不需要突出显示的链接。并且a用于外部链接。
